# INIT-2026Q2-improved-clarify — Улучшение вопросов Clarify

**Приоритет:** Средний  
**Период:** 2026Q2  
**Статус:** Proposed  
**Owner:** TBD

## Контекст
Частая проблема: clarify задаёт “лишние” вопросы и размазывает диалог по итерациям.
Нужно сокращать clarify-итерации и повышать осмысленность вопросов.

## Цель
- Генерировать один осмысленный clarify-вопрос (LLM-assist)
- Определять список реально недостающих полей
- Baseline уточняет только необходимые атрибуты

## Scope
- `missing_fields` (internal representation)
- LLM assist для формулировки одного вопроса
- Правила baseline:
  - уточнять только missing_fields
  - не спрашивать то, что уже известно/однозначно
- Метрика: снижение clarify-итераций

## Out of scope
- Диалоговый менеджер уровня UI
- Долгая память

## Acceptance criteria
1. Clarify становится короче по числу итераций на golden dataset
2. Вопросы не содержат лишней информации и не раскрывают raw input в логах
3. Логирование/метрики есть

## Deliverables
- missing_fields механизм
- improved clarify question pipeline
- тесты/кейсы
