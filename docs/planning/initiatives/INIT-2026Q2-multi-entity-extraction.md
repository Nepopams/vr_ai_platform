# INIT-2026Q2-multi-entity-extraction — Поддержка множественных сущностей

**Приоритет:** Средний  
**Период:** 2026Q2  
**Статус:** Proposed  
**Owner:** TBD

## Контекст
Shopping-команды часто содержат списки: “молоко, хлеб и бананы”, плюс количества/атрибуты.
Нужно сделать extraction устойчивым и совместимым с текущими схемами.

## Цель
- Надёжно поддержать списки покупок
- Поддержать количества/единицы/простейшие атрибуты
- Не поломать контрактную совместимость

## Scope
- Extraction для shopping lists:
  - list items (N)
  - quantity + unit (если явно)
  - attribute hints (например “обезжиренное”, “без сахара”)
- Подготовка структуры данных так, чтобы укладываться в существующие schemas
- Тест-кейсы + golden dataset расширение

## Out of scope
- Сложные NLP-онтологии продуктов
- Персональные предпочтения пользователя (память)
- Выполнение задач

## Acceptance criteria
1. На тест-наборе корректно распознаются multi-item команды
2. Контрактные схемы не меняются (или changes строго совместимы)
3. Есть тесты и примеры

## Deliverables
- Обновлённые extraction функции
- Расширенный golden dataset
- Документация по supported patterns
