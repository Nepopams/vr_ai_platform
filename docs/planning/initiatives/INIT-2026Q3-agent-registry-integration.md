# INIT-2026Q3-agent-registry-integration — Интеграция агентной платформы v0

**Приоритет:** Низкий  
**Период:** 2026Q3  
**Статус:** Proposed  
**Owner:** TBD

## Контекст
Нужен единый реестр способностей и управляемое подключение “внутренних агентов”
без изменения ядра. Это позволит расширять эксперименты безопасно.

## Цель
- Подключить agent_registry и раннер для внутренних агентов (baseline shopping, clarify)
- Формализовать через ADR (например ADR-005)

## Scope
- agent_registry интеграция (capabilities, metadata)
- runner integration (без исполнения доменных действий)
- Обновление документации и ADR
- Эксперименты только под флагом

## Out of scope
- Marketplace внешних агентов
- Автономные многошаговые агенты

## Acceptance criteria
1. Реестр способностей доступен платформе
2. Подключение/отключение агентов управляется конфигом/флагами
3. ADR фиксирует границы и риски

## Deliverables
- agent_registry integration layer
- ADR-005
- docs updates
